USER GUIDE

How Does Someone Use Vocation By Location?

Vocation By Location is a REST Service.

The user will need to send HTTP GET requests to use the service.
It does not matter what language they use to send these requests.
These requests will be secure (https).

The responses from this service will come back as JSON Objects.

No Authentication of any kind is needed to use the service.

Our service is located here: https://ourcompany.com
From here you can send requests and receive responses to those requests.

You Send:
A session will need at least two http get requests through our service to get the desired results.

1. An HTTP GET request looking for a selected industry.
An http Industry Get Request for the service: (https://ourcompany.com/api/industries)

2. An HTTP GET request looking for the places where that industry is located.
An http Place Get Request for the service: (https://ourcompany.com/api/places?industry=(industry id))

You Receive:
A session will return a list of JSON objects for each request.

1. A JSON Object that contains the industry name, the average wage, and its corresponding id.
JSON Object returned for Industry: {industry name, average wage, id}.

2. A JSON Object that contains the place name and its corresponding state, and population.
JSON Object returned for Place: {place name, companies per 1,000 people, state, population}.
